# admin-hth-resource

通用 HTML 页面归档与发布仓库。

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面不针对任何特定的域名或网站，仅作为静态资源的集合，便于统一管理和分发。

## 目录结构

```
admin-hth-resource/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── docs/           # 相关文档或说明文件
└── README.md       # 本文件
```

- `pages/`：各 HTML 页面按需组织存放，可包含子目录。
- `assets/`：与页面配套使用的资源文件，保持与页面的相对引用关系。
- `docs/`：额外的说明文档或使用指南。

## 页面归档说明

- 所有页面均为独立 HTML 文件，不依赖外部数据库或服务端环境。
- 页面引用资源（如样式表、脚本、图片）建议使用相对路径，以确保在本地或静态托管环境中正常访问。
- 每个页面应具有自包含性，尽量减少对外部源的依赖。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接使用浏览器打开 `pages/` 目录下的 HTML 文件即可预览。
3. 如需部署到静态托管服务，可将整个仓库内容上传至服务根目录。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并确保资源引用路径正确。
- 修改已有页面时，请保持文件结构稳定，避免破坏已有引用关系。
- 不建议在页面中嵌入敏感信息或动态内容，保持静态页面的简洁性。
- 如页面需要更新，请提交 Pull Request 或直接推送至主分支。
